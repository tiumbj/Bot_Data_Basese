- ยืนยัน mandatory TF coverage

Step 3 - Registry truth
- research outputs ต้อง map เข้า candidate/approved/deprecated ได้จริง

Step 5 - Intelligent DB runtime truth
- ingest research + feature + registry เข้า DB กลาง
- ยืนยัน query path ที่ bot ใช้จริง

Step 6 - Realtime and MT5 loop truth
- bot query DB
- signal engine ประเมินตาม policy กลาง
- MT5 ส่งผล execution กลับเข้า DB

Step 7 - Weekly intelligence truth
- rerun / rerank / refresh / demote / publish เป็น routine

8) ANTI-DEVIATION DECISION RULES
====================================================================
ก่อนทำงานใด ๆ ให้เช็คตามนี้

Rule 1
ถ้างานนั้นไม่เพิ่มความชัดของ central platform layer ใด layer หนึ่ง
=> อย่าทำก่อน

Rule 2

=> เสี่ยงหลงทาง

Rule 3
ถ้างานนั้นทำให้ bot ตัดสินเองโดยไม่ query จากฐานกลาง
=> ผิดทิศทาง

Rule 4
ถ้างานนั้นข้าม micro exit หรือข้าม TF บางตัวที่ล็อก
=> ผิด project truth

Rule 5
ถ้างานนั้นอ้างว่า production-ready แต่ยังเป็นแค่ spec txt
=> ห้ามนับเป็น runtime completion

Rule 6
ถ้างานนั้นเพิ่ม research quantity แต่ไม่เพิ่ม standardized evidence / promotion logic
=> ยังไม่ถือว่าปิด production gap
