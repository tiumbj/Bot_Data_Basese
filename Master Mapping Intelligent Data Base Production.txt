4) Production Progression Checklist

ใช้ checklist นี้เป็น “anti-deviation control”

Phase 0 — Concept Lock

ต้องยืนยันก่อนเริ่มทุกงาน:

 research TF = M1,M2,M3,M4,M5,M6,M10,M15,M30,H1,H4,D1
 execution TF แยกจาก research TF
 micro exit = mandatory base layer
 backtest core = custom + controllable
 intelligent database = central system
 เป้าหมาย = profit + survivability ไม่ใช่ zero-trade perfection
Phase 1 — Canonical Data Ready
 M1 source กลางตัวเดียว
 สร้างทุก TF ครบ
 parquet cache ครบทุก TF
 integrity checks ผ่าน

Phase 5 — Intelligent Database Runtime

Phase 6 — Realtime Decision Layer

Phase 7 — MT5 Execution Integration

Phase 8 — Weekly Intelligence Loop

Phase 9 — Production Governance

 every promotion backed by evidence
5) Deviation Guard Rules

ใช้เป็นกฎคุม dev ทุกครั้ง

ห้ามเบี่ยง 1

ห้ามลดโปรเจกต์เหลือแค่ bot ยิงออเดอร์
เพราะ concept จริงคือ central intelligence platform

ห้ามเบี่ยง 2

ห้ามใช้ VectorBT เป็น core production engine
ใช้ได้เฉพาะ research/support layer เท่านั้น

ห้ามเบี่ยง 3

ห้ามข้าม micro exit
เพราะนี่คือ base layer บังคับ

ห้ามเบี่ยง 4

ห้ามวิจัยเฉพาะ TF ที่ชอบ
ต้อง cover TF ที่ล็อกครบ

ห้ามเบี่ยง 5

ห้ามเลือก winner จากกำไรอย่างเดียว
ต้องใช้ multi-metric + robustness

ห้ามเบี่ยง 6

ห้ามปล่อย strategy เข้า live โดยไม่มี registry/approval
ต้องผ่าน central winner registry ก่อน

ห้ามเบี่ยง 7

ห้ามให้ bot ตัดสินโดยไม่มีฐานข้อมูลกลาง
bot ต้อง query จาก intelligent database ไม่ใช่สร้าง truth เอง

ห้ามเบี่ยง 8

ห้าม frozen model ยาวโดยไม่มี weekly refresh
เพราะ concept นี้ต้องมี weekly intelligence loop
