SECTION 1: FINAL PROJECT SPLIT
================================================================

[A] INTELLIGENT DATABASE PLATFORM
หน้าที่:
- รับข้อมูล MT5
- canonicalize OHLCV
- dedup / quarantine / continuity validation
- rebuild timeframe datasets
- สร้าง feature store
- run backtest / validation / walk-forward
- สร้าง labels / case memory / retrieval index
- publish active package / weekly intelligence snapshot
- เปิด SDK/API ให้ bot อื่นใช้
- รับ live feedback กลับเข้าระบบ

[B] BOT RUNTIME PLATFORM
หน้าที่:
- อ่านข้อมูลจาก platform กลางผ่าน SDK/API เท่านั้น
- ขอ signal context / active package / similar cases / regime snapshot
- ประเมิน candidate setup
- ส่ง context ให้ local LLM
- รับ final confirm
- ส่งคำสั่งเข้า MT5
- monitor order
- ส่ง feedback กลับฐานข้อมูลกลาง

LOCKED RULE:
Bot ห้ามอ่าน Parquet, DuckDB internal tables, registry files, quarantine files, or feature files โดยตรง
Bot ต้องผ่าน SDK หรือ FastAPI เท่านั้น

2. validate required paths / env
3. create folder tree
4. initialize logging
5. create/open DuckDB registry
6. apply migrations
7. verify storage roots
8. verify mandatory config sections
9. write init run row to registry
10. print bootstrap summary

SUCCESS CRITERIA:
- DuckDB file exists
- schema version table initialized
- folder tree exists
- config resolved successfully
- run registry has INIT_SUCCESS row

================================================================
SECTION 6: WEEKLY MASTER FLOW
================================================================

COMMAND TARGET:
python -m idp.jobs.run_weekly_cycle

ORDER:
1. start master run
2. weekly_ingest_job
3. rebuild impacted TF datasets
4. refresh feature store
5. weekly_backtest_job
6. refresh labels
7. refresh memory / retrieval index
8. weekly_publish_job
9. write intelligence snapshot
10. mark active aliases
11. close master run
12. write docs step update entry

FAILURE RULES:
- ingest fail => stop full cycle
- backtest fail => do not publish new active package
- publish fail => keep prior active alias unchanged
- retrieval refresh fail => publish allowed only if last good retrieval snapshot exists
